## 核心模块详解

### 1. tiles.js — 牌面定义

定义麻将牌的基本属性和工具函数。

**牌的表示:**
```js
{ id: 42, type: 'w', value: 3 }  // 三万
{ id: 120, type: 'f', value: 2 } // 南风
{ id: 124, type: 'd', value: 1 } // 红中 (赖子)
```

**牌型分类:**
| type | 含义 | 数值范围 | 数量 |
|------|------|---------|------|
| `w` | 万 | 1-9 | ×4 = 36 |
| `t` | 筒 | 1-9 | ×4 = 36 |
| `b` | 条 | 1-9 | ×4 = 36 |
| `f` | 风 (东南西北) | 1-4 | ×4 = 16 |
| `d` | 箭 (中发白) | 1-3 | ×4 = 12 |

总计 136 张牌，`d1`（红中）为赖子。

**关键函数:**
- `createDeck()` → 生成 136 张牌
- `shuffle(arr)` → Fisher-Yates 洗牌
- `tileKey(tile)` → 牌的唯一键 (如 `"w3"`, `"f2"`)
- `isLaizi(tile)` → 判断是否为红中赖子
- `tileSort(a, b)` → 排序比较函数

### 2. huChecker.js — 胡牌检测

递归回溯算法，支持赖子（红中）作为百搭牌。

**算法流程:**

```
canHu(tiles, meldCount)
  │
  ├─ 统计赖子数量，将其从手牌中分离
  │
  ├─ 尝试所有可能的雀头（将）:
  │   ├─ 2 张赖子作对
  │   ├─ 1 赖子 + 1 真牌作对
  │   └─ 2 张相同真牌作对
  │
  └─ 对剩余牌尝试分解为面子:
      canFormMelds(counts, laizi, remaining)
        │
        ├─ 刻子: 3 张相同 (可用 0-2 赖子补)
        ├─ 顺子: 3 张连续 (仅万/筒/条，可用赖子补缺)
        ├─ 3 赖子组成的面子
        └─ 递归直到所有牌被消耗
```

**关键设计:**
- 面子分解时始终取剩余牌中最小的一张，确保不遗漏
- 顺子尝试三种位置关系（当前牌作为顺子的第一张、第二张、第三张）
- 赖子数量约束保证不会过度使用

**复杂度:** 最坏情况 O(3^n)，其中 n 为手牌种类数（≤ 34）。实际运行中由于 early return 和约束剪枝，性能远优于理论上限。

### 3. Game.js — 游戏引擎

核心状态机，管理整个麻将游戏流程。

**游戏状态:**

```
idle ──start()──→ playing ──hu/流局──→ finished
  ↑                                        │
  └──────────── reset() ←──────────────────┘
```

**回合流程:**

```
┌─────────────────────────────────────────────┐
│              玩家回合                         │
│                                              │
│  1. 摸牌 (_drawForPlayer)                    │
│     ├─ 牌墙空? → 流局                        │
│     └─ 摸一张牌到手牌                         │
│                                              │
│  2. 检查自摸操作 (_processCurrentTurn)        │
│     ├─ 可胡? → 提示自摸胡                     │
│     ├─ 可暗杠? → 提示暗杠                     │
│     ├─ 可加杠? → 提示加杠                     │
│     └─ 否则 → 等待出牌                        │
│                                              │
│  3. 出牌 (_doDiscard)                         │
│     └─ 牌加入弃牌堆                           │
│                                              │
│  4. 检查其他玩家响应 (_checkDiscardActions)    │
│     ├─ 胡 > 杠 > 碰 (优先级)                  │
│     ├─ 有响应 → 等待所有人回应 → _resolveActions│
│     └─ 无响应 → 下一玩家摸牌                   │
│                                              │
│  5. 动作处理                                   │
│     ├─ 碰: 拿牌 + 手中取 2 张 → 副露 → 出牌   │
│     ├─ 明杠: 拿牌 + 手中取 3 张 → 副露 → 摸牌  │
│     └─ 胡: 游戏结束                            │
└─────────────────────────────────────────────┘
```

**动作优先级系统:**

当一张牌被打出后，按以下优先级处理多个玩家的响应:
1. **胡** — 最高优先级，立即处理
2. **杠** — 次高
3. **碰** — 最低

同优先级时，按距离出牌者的顺序（下家 > 对家 > 上家）决定。

机器人即时决策，人类玩家有 12 秒响应时间，超时自动过。

**发牌流程:**
1. 洗牌生成 136 张
2. 每人 4 张 × 3 轮 = 每人 12 张
3. 每人再摸 1 张 = 每人 13 张
4. 庄家额外摸 1 张 = 庄家 14 张
5. 牌墙剩余: 136 - 52 - 1 = 83 张

### 4. ai.js — AI 机器人

基于启发式评分的简单 AI。

**决策策略:**

| 场景 | 策略 |
|------|------|
| 可胡 | 始终胡牌 |
| 可杠 | 始终杠 |
| 可碰 | 大概率碰 (70%+，有对子时 100%) |
| 出牌选择 | 丢弃「 usefulness 评分」最低的牌 |

**出牌评分算法:**
- 赖子（红中）: 1000 分（绝不丢弃）
- 手中有相同牌: +15 × 张数
- 同花色相邻牌: +10
- 同花色间隔 1 张: +4
- 孤张风/箭牌: -5
- 随机扰动: 0-3（避免可预测性）

### 5. index.js — 服务端入口

**职责:**
- Express 提供静态文件服务
- WebSocket 处理实时通信
- 房间管理（创建/加入/销毁）
- 消息路由（客户端消息 → Game 实例方法）

**房间管理:**
```
rooms = {
  "AB3F": Game实例,
  "XK72": Game实例,
  ...
}
connections = Map {
  ws1 → { playerId, roomId, name },
  ws2 → { playerId, roomId, name },
  ...
}
```

## WebSocket 消息协议

### 客户端 → 服务端

| 消息类型 | 字段 | 说明 |
|---------|------|------|
| `createRoom` | `name` | 创建房间 |
| `joinRoom` | `roomId`, `name` | 加入房间 |
| `addBot` | — | 添加机器人 |
| `startGame` | — | 开始游戏 |
| `discard` | `tileId` | 出牌 |
| `action` | `action` (hu/gang/peng/pass) | 响应操作 |
| `restart` | — | 再来一局 |

### 服务端 → 客户端

| 消息类型 | 说明 |
|---------|------|
| `connected` | 连接成功，返回 playerId |
| `roomJoined` | 加入房间成功 |
| `playerJoined` | 新玩家加入 |
| `gameStart` | 游戏开始，含初始手牌 |
| `drawTile` | 摸到一张牌 |
| `playerDraw` | 其他玩家摸牌（隐藏牌面） |
| `playerDiscard` | 有人出牌 |
| `turnInfo` | 回合信息 |
| `actionRequest` | 请求玩家操作（碰/杠/胡/出牌） |
| `handUpdate` | 手牌更新 |
| `peng` / `gang` | 碰/杠事件 |
| `gameOver` | 游戏结束（含所有人手牌） |
| `gameReset` | 游戏重置 |
| `error` | 错误消息 |

## 前端架构

单页应用 (SPA)，三个视图切换:

1. **Lobby** (`#lobby`) — 创建/加入房间
2. **Room** (`#roomView`) — 等待玩家
3. **Game** (`#gameView`) — 游戏桌面

**桌面布局 (以玩家 0 视角为例):**

```
           ┌─── 对家 (player 2) ───┐
           │  [牌背] [副露]         │
     ┌─────┤                       ├─────┐
     │     │                       │     │
  左家  │     │    ┌───────────┐    │  右家
 (p3)  │     │    │  弃牌区    │    │  (p1)
     │     │    │  (牌河)     │    │     │
     │     │    └───────────┘    │     │
     └─────┤                       ├─────┘
           │  [我的手牌]  [副露]    │
           └─── 自己 (player 0) ───┘
           [ 碰 ] [ 杠 ] [ 胡 ] [ 过 ]
```

位置映射通过 `getPlayerPositions()` 动态计算，确保每个玩家都看到自己在底部。
